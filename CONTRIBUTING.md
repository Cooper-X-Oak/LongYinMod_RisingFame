# Contributing

感谢你愿意一起完善 `RisingFame`。

这个项目的优先级：

1. 稳定启动
2. 低运行负担
3. 实际入账生效
4. 文档清晰

如果一个改动会让功能更多，但更容易导致启动慢、卡顿、闪退或兼容性变差，那么默认不接受。

## 项目边界

当前仓库以稳定精简版为准。

- 支持环境以 `README.md` 为准
- 超出固定环境的兼容性，不作为当前版本的默认支持目标
- 如果改动只让显示变化、没有改到真实入账，默认视为不合格

## 欢迎的贡献

- 修复真实不生效的问题
- 降低启动风险、减少卡顿、减少不必要 Hook
- 改进 README、QA、release 文案和 issue 模板
- 补充清晰、低负担、边界明确的倍率功能

## 默认不鼓励的方向

- 运行时大范围反射扫描
- 注入额外 Unity 组件
- 高频日志
- 只改显示、不改真实入账
- 增加大量可选配置、额外依赖或复杂初始化流程

## 提交 Issue 前

- 请优先使用仓库现有模板
- 打不开 / 不生效 / 很慢：尽量附上 `BepInEx/LogOutput.log`、游戏根目录截图、`BepInEx` 压缩包完整文件名
- 功能建议：请说明痛点、影响范围，以及是否可能带来启动负担或兼容风险

## 提交 Pull Request 时

请尽量保持改动小而明确，并说明：

1. 改了什么
2. 为什么要改
3. 实际测试了什么
4. 是否同步更新了 README / CHANGELOG / release 文案

推荐自检：

- 只装固定版本 `BepInEx` 时，游戏可以正常启动
- 放入插件后，插件可以正常加载
- `=` 开关可以稳定切换
- 相关倍率改动作用于真实入账，而不只是显示文本
- 没有新增高频日志或明显增加启动负担

## 文档口径

为了减少混乱，公开文档尽量统一使用下面这些说法：

- `武学经验（读书 / 实战）`
- `技艺经验`
- `好感增长`
- `门派&官府功绩`
- `阶位每 +1，倍率 +0.5`

如果改动影响玩家理解，请同步更新：

- `README.md`
- `CHANGELOG.md`
- `.github/RELEASE_TEMPLATE_FULL.md`
- `.github/RELEASE_TEMPLATE_SHORT.md`

总原则：少而稳、清楚可验证、功能边界明确。
